# Журнал работ на строительном объекте

Тестовое задание: внутренний инструмент для фиксации выполненных работ на строительном объекте. Прораб может вести ежедневный журнал, фильтровать записи по датам, добавлять, редактировать и удалять работы.

## Что реализовано

- Список записей журнала в табличном виде: дата, вид работ, объем с единицей измерения, исполнитель и комментарий.
- Фильтрация записей по диапазону дат.
- Добавление записи через форму с валидацией обязательных полей.
- Удаление записи.
- Редактирование существующей записи.
- Справочник видов работ и возможность добавить свой вид работ прямо из формы.
- Сводка по выбранному периоду: количество записей, общий объем, число исполнителей.
- Состояния загрузки, ошибок и пустого списка.
- Адаптивный интерфейс в стиле внутреннего B2B-сервиса: широкие окна, узкие окна, планшет и мобильный вид.

## Технические требования

Выполнено:

- Фронтенд: React + TypeScript.
- Данные хранятся в базе данных SQLite, не в памяти и не в файле JSON.
- Фронтенд взаимодействует с бэкендом через REST API.
- Бэкенд: Node.js + Express.
- Валидация данных есть на клиенте и на сервере.
- Пользовательские виды работ сохраняются в справочник в базе данных.
- Есть автоматические тесты для API и модели формы.
- Есть браузерная проверка основного пользовательского сценария.

## База данных

При запуске создается SQLite-база:

```text
data/work-journal.sqlite
```

Таблицы:

- `work_types` — справочник видов работ.
- `journal_entries` — записи журнала выполненных работ.

Папка `data/` добавлена в `.gitignore`, чтобы локальная база не попадала в репозиторий.

## API

```text
GET    /api/health
GET    /api/work-types
GET    /api/entries?from=YYYY-MM-DD&to=YYYY-MM-DD
POST   /api/entries
PUT    /api/entries/:id
DELETE /api/entries/:id
```

Пример тела для создания или обновления записи:

```json
{
  "workDate": "2026-05-28",
  "workTypeId": 2,
  "quantity": 24,
  "unit": "м2",
  "performer": "Бригада Монолит-2",
  "comment": "Ось А-Г, этаж 3"
}
```

Если нужно создать свой вид работ, вместо `workTypeId` можно передать `workTypeName`:

```json
{
  "workDate": "2026-05-28",
  "workTypeName": "Гидроизоляция швов",
  "quantity": 42,
  "unit": "м.п.",
  "performer": "ООО Изолстрой",
  "comment": ""
}
```

